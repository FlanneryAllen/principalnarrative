# Applied Narrative

This directory contains the organizational context and narrative foundation for Principal AI. The Narrative Agent uses this Git-native storage to maintain and evolve the company's strategic context, decisions, and brand voice.

## Purpose

The Applied Narrative system serves as:
- **Single Source of Truth**: Centralized organizational knowledge
- **Version Controlled**: All changes tracked in Git
- **Agent-Accessible**: Structured for AI agent consumption and updates
- **Human-Readable**: Markdown format for easy collaboration

## Structure

```
applied-narrative/
├── vision.md                 # Company vision and mission
├── priorities.md             # Current strategic priorities
├── brand-voice.md            # Brand voice and communication guidelines
├── customer-pain-points.md   # Customer problems we solve
├── decisions/                # Architecture Decision Records (ADRs)
├── technical-context/        # Technical architecture and context
└── .meta/                    # Metadata and schemas
```

## Usage

### For Humans
- Edit files directly to update organizational context
- Use frontmatter to add metadata
- Commit changes with descriptive messages
- Review history to understand evolution

### For Narrative Agent
- Read context files to understand organizational state
- Update files based on new decisions or insights
- Create new decision records
- Maintain index and cross-references

## Document Types

### Core Context Files
- **vision.md**: Long-term vision, mission, and values
- **priorities.md**: Current quarterly/annual priorities
- **brand-voice.md**: Tone, style, and messaging guidelines
- **customer-pain-points.md**: Problems we solve and target audience

### Directories
- **decisions/**: Architecture Decision Records and significant choices
- **technical-context/**: System architecture, tech stack, patterns

### Meta
- **schema.json**: JSON schema definitions for all document types
- **index.json**: Cross-references and relationships between documents

## Best Practices

1. **Keep It Current**: Update documents as context changes
2. **Be Specific**: Avoid vague statements; include concrete examples
3. **Track Evolution**: Don't delete old content; use versioning
4. **Link Decisions**: Reference related documents in frontmatter
5. **Review Regularly**: Schedule quarterly reviews of core context

## Contributing

When updating narrative documents:
1. Update the `lastUpdated` field in frontmatter
2. Add your changes to the `changelog` section
3. Commit with descriptive message explaining the "why"
4. Tag related commits if making coordinated changes
