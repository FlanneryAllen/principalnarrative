Quick Fix: One-Click Code Improvements

PL
Priya Lakshmi
Product Manager · October 12, 2024 · 5 min read

Code review has always been a two-step dance: AI (or a human) spots an issue, then the developer goes back to their editor to fix it. We wondered: what if we could collapse those steps?

Today we're launching Quick Fix—one-click code improvements that apply CodePilot's suggestions directly to your PR. No context-switching, no copy-paste, just click and the fix is applied.

✨ Quick Fix is Live

Accept suggestions with one click. Watch changes apply automatically.

The Problem: Friction Between Review and Fix

When CodePilot (or a human reviewer) suggests a change, the traditional workflow looks like this:

  1. Read the suggestion in the PR
  2. Context-switch to your editor
  3. Find the file and line number
  4. Make the change manually
  5. Commit and push
  6. Return to the PR to see if you did it right

For small fixes—adding a null check, fixing a typo, reordering imports—this is pure overhead. The suggestion tells you exactly what to change. Why should you have to manually type it?

The Solution: Executable Suggestions

With Quick Fix, applicable suggestions come with a button. Click it, and CodePilot applies the change directly to your branch.

Example: Quick Fix in Action

⚡ Quick Fix Available
This async function doesn't await the database call, which means errors won't be caught and data may not be saved before the function returns. Add await to ensure the operation completes.
- db.saveUser(newUser);
+ await db.saveUser(newUser);

Click the button. Two seconds later, the change is committed to your branch. No editor switching, no manual typing.

What Quick Fix Handles

Quick Fix works for deterministic changes where the fix is unambiguous. We've enabled it for:

🔧 Common Bugs

Missing null checks, unhandled promises, off-by-one errors

📝 Style & Formatting

Import ordering, spacing, naming conventions

🔒 Security Fixes

Parameterized queries, input escaping, secure random generation

♻️ Simple Refactors

Extract constants, rename variables, add error handling

📚 Documentation

Adding JSDoc comments, updating outdated docs

🧪 Test Improvements

Missing assertions, test descriptions, setup/teardown

How We Decide What's "Quick Fixable"

Not every suggestion can be auto-applied. Quick Fix is only offered when:

  • The fix is deterministic: There's one correct way to make the change
  • Context is sufficient: We understand the surrounding code well enough to apply the change safely
  • Impact is localized: The change affects a small, well-defined area
  • Risk is low: The change is unlikely to introduce new issues

Architectural changes, business logic decisions, and anything requiring human judgment? Those still need human review. Quick Fix handles the routine, mechanical changes.

Developer Experience: Staying in Flow

The time savings are real, but the bigger win is preserving flow state. Context-switching from PR review back to your editor and back to the PR breaks concentration.

"Quick Fix lets me process review feedback without leaving the PR. I can accept the trivial fixes immediately and focus my mental energy on the architectural feedback that actually requires thought."

— Beta tester feedback

When reviewing your own PR after CodePilot's first pass:

  1. Scan the suggestions
  2. Quick Fix the obvious ones (styling, null checks, etc.)
  3. Think deeply about the architectural or logic suggestions
  4. Request human review for complex decisions

The result: human attention goes where it's actually needed.

Safety & Control

Letting AI write directly to your codebase requires trust. Here's how we built Quick Fix safely:

Quick Fix Safety Features

  • Always creates a new commit: Every Quick Fix is a separate commit on your branch—full git history and easy rollback
  • Shows the diff first: You see exactly what will change before clicking apply
  • Respects your workflows: Triggers your CI/tests just like manual commits
  • Opt-in per suggestion: Each suggestion requires explicit approval
  • No forced changes: You can always dismiss and make the change manually

And if Quick Fix ever applies a change you don't like? Standard git revert, or just push your own correction.
